# Domain Docs — Consumer Rules

## Layout

Single-context: one `CONTEXT.md` at the repo root, one `docs/adr/` directory.

## Reading rules

- Read `CONTEXT.md` before proposing architectural changes — it defines the domain language.
- Read relevant ADRs in `docs/adr/` before revisiting a past decision.
- Never modify `CONTEXT.md` or ADRs without explicit user approval.
- ADR format: `docs/adr/NNNN-<slug>.md` using the standard ADR template.
